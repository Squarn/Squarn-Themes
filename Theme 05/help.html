HELP!

Squarn theme help & documentation

POLICY I know what it's like to learn how to handle HTML and CSS, what it's like to be mystified by a bit of code, so I will help and provide documentation to the best of my ability. In general, I will help with vanilla errors (I can't test every case, and you can only go so dynamic) and adding functionality that wasn't reaching enough to make it into the tumblr customization menu. I won't write vast swathes of personal edits for you, but if you're asking for help from a learning standpoint, I will at least give you pseudocode or a point in the right direction. If you want to know how I did something in a theme, I will be more than happy to help.

ASKING A QUESTION Asking off anon makes things loads easier, because I can inspect your theme's code directly and try to find the problem. I'm really not going to judge you, for anything. If you must use anon, PLEASE be as specific as possible. Please don't ask me to fix problems in code you already edited. Refer to the "policy" section above for asking how to edit. If I've made a blunder, PLEASE tell me.

GISTS I host my explanations and code snippets on Github's Gist feature so that they're easily editable, you don't have to dig through tumblr to find them, and because Github flavored Markdown is rather smashing. Find all of my gists here.

GITHUB I host all of my themes and their documentation on Github.com, a version controlled code hosting site where many devs nurture their code repositories and projects. It allows me to make edits on my desktop (using Sublime Text) and easily push these edits (commits) to the Github server without mucking around copying and pasting to pastebin. With version control, you can even keep track of edits and fork/merge files. Learn more here.
