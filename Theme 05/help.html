POLICY I know what it's like to learn how to handle HTML and CSS, what it's like to be mystified by a bit of code, so I will help and provide documentation to the best of my ability. In general, I will help with vanilla errors (I can't test every case, and you can only go so dynamic) and adding functionality that wasn't reaching enough to make it into the tumblr customization menu. I won't write vast swathes of personal edits for you, but if you're asking for help from a learning standpoint, I will at least give you pseudocode or a point in the right direction. If you want to know how I did something in a theme, I will be more than happy to help.

GITHUB I host all of my themes and their documentation on Github.com, a version controlled code hosting site where many devs nurture their code repositories and projects. It allows me to make edits on my desktop (using Sublime Text) and easily push these edits (commits) to the Github server without mucking around copying and pasting to pastebin. With version control, you can even keep track of edits and fork/merge files. Learn more here.

GISTS I host my explanations and code snippets on Github's Gist feature so that they're easily editable, you don't have to dig through tumblr to find them, and because Github flavored Markdown is rather smashing. Find all of my gists here.

EDITOR: Sublime Text — I've only been using this text editor for about a month but it's already changed so much about how I work. It uses an addon called Package Control to add plugins and third-party syntax highlighting AND UNF. It's a sexy, sexy program and I wouldn't let anyone use anything else.

My favorite plugin is called Emmet, which basically lets you use shorthand for writing all of your html tags. On a large scale, you could write html>(head>title+style)+body>(header>navigation>ul>li*5>a)+(main>article*5)+footer, hit TAB, and it would expand to the base for an ENTIRE layout, something that would take me many minutes to write out. It looks daunting, but it's really easy to get into once you read the syntax doc. On a small scale, I can simply write out "div.soup", hit TAB, and it will expand to ~div class="soup~/div~. It really saves so much time once you get used to it.

The third party color themes for Sublime Text are really nice. I recommend the theme Spacegray, which uses the base 16 color schemes ocean dark, ocean light, and eighties. I recommend using Package Control to install the Base 16 color schemes as well. Other nice ones are Afterglow and Glacier. Browse more on wbond.net.

STANDARDS I try to keep up to date and use the most current WC3 standards of HTML5 and CSS3. I see people using old doctyes while using new css3 techniques and it's a bit mind-boggling. I am by NO means perfect, however, and I will never claim to be, which is why I'm always reading articles and docs.
